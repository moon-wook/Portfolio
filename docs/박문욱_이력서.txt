자기소개서

AI 모델 개발 및 데이터 분석 중심의 엔지니어
[AI로 더 나은 세상을 만드는 엔지니어의 성장기]
평소 기술이 사람과 사회를 어떻게 변화시킬 수 있을까에 대해 많은 관심을 가져왔습니다.
단순한 기술 습득을 넘어, AI를 통해 더 나은 세상에 기여하고 싶다는 목표가 제 학습의 출발점이었습니다.

대학교 재학 중 처음으로 딥러닝과 컴퓨터 비전을 접하면서 AI의 가능성에 매료되었습니다.
직접 CNN 기반 이미지 분류와 YOLO 기반 객체 검출 프로젝트를 수행하며, 기술이 인간의 불편함을 줄이고 안전을 높일 수 있음을 체감했습니다.
이 과정에서 AI 개발자로서의 방향성을 확실히 다지게 되었고, 보다 깊은 실무 역량을 쌓기 위해 KT AIVLE School 부트캠프에 도전했습니다.

부트캠프에서는 데이터 분석, 머신러닝, 딥러닝, NLP 등 산업 현장에서 활용되는 기술을 심도 있게 학습했습니다.
총 7개의 미니 프로젝트와 1개의 빅 프로젝트를 수행하며 데이터 전처리 모델링 성능 평가까지 전 과정을 경험했고,
특히 시계열 예측과 BERT 기반 감정분석 모델 구현을 통해 다양한 데이터 유형을 다루는 실무 감각을 익혔습니다.

현재는 WI.Plat에서 IoT 센서로 수집된 음향 데이터를 활용한 CNN 기반 누수 예측 모델 개발을 담당하고 있습니다.
데이터 전처리부터 특징 추출(FFT·MEL), 모델 설계·학습·평가까지 전 과정을 수행하며,
AI 기술이 실제 산업 현장의 문제 해결과 사회적 가치 창출로 이어질 수 있음을 실감했습니다.

앞으로도 AI 기술을 단순한 기능이 아닌, 사람과 환경을 이롭게 하는 솔루션으로 발전시켜 나가는 개발자로 성장하겠습니다.

[성격]
장점 : "적극적인 소통과 팀워크"
저는 팀원들과 활발히 소통하며 협업의 시너지를 만들어내는 것을 즐깁니다. 의견 충돌 시 먼저 팀원들을 모아 각자의 생각을 듣고, 적극적인 질문을 통해 모두가
자유롭게 의견을 나눌 수 있는 분위기를 조성합니다. 협업은 함께 만들어가는 과정이라 믿기에, 팀 내에서 먼저 다가가 대화를 이끌고 모두의 목소리가 반영되도
록 하는 역할을 자연스럽게 맡고 있습니다.

단점 : "멀티태스킹의 한계와 개선 노력"
여러 업무를 동시에 처리할 때 집중력이 분산되는 경향이 있습니다. 특히 소통을 중요하게 여기다 보니 여러 요청이 동시에 들어올 때 우선순위 설정에 어려움을
겪기도 했습니다.
이를 개선하기 위해 업무 우선순위 매트릭스로 작업 순서를 명확히 정하고, 팀원들에게 집중 업무와 일정을 투명하게 공유하며 협조를 구하고 있습니다. 한 번에
하나의 업무에 집중한 후 즉시 결과를 공유하고 다음 단계를 논의하는 방식으로 소통의 흐름을 유지합니다.

현재는 '빠르게 여러 가지'보다 '확실하게 하나씩, 그리고 함께' 완수하는 것을 저만의 업무 스타일로 발전시켰습니다.

[직무 관련 프로젝트 경험]

1) WI.Plat 누수 판별 시스템 (2025.02 ~ 현재)
문제정의: 상수도망의 미세 누수를 사람의 청음 의존 없이 AI로 자동 판별해야 하는 현장 문제를 해결.
역할: CNN 기반 누수 판별 모델 E2E 개발(전처리특징추출설계·학습·검증)
기술: FFT·MEL, TensorFlow/Keras, Librosa, Python/Pandas/NumPy
주요기여: IoT 음향 데이터 파이프라인을 구축하고, 다양한 소음 환경에 대응하기 위해 데이터 증강·임계값 조정 적용.
성과/피드백: 다양한 소음 환경에서도 안정적 정확도 확보, 모델의 재현성·신뢰성 향상, AI가 산업 현장의 실질적 문제를 해결할 수 있다는 실용적 가치와 데이터
품질 관리의 중요성을 체감.

2) 발전소 배관 누수 진단 (2025.05 ~ 현재)
문제정의: 발전소 내 배관 및 밸브 구간의 고소음 환경 속 미세 누수 조기 탐지 필요성 해결.
역할: 발전소 전용 누수 예측 모델 고도화 및 현장 적응화
기술: 노이즈 분석·제거(고주파/저주파), CNN 재학습, 리스크 스코어링
주요기여: 수집–정제–특징화 파이프라인을 개선해 잡음 민감도·판별 안정성, 현장 점검에 리스크 우선순위 리스트 제공.
성과/피드백: 도메인 특성이 모델 성능을 크게 좌우함을 확인, 환경 적응형 학습과 데이터 거버넌스 구축의 중요성을 인식.

3) HI WIPLAT – 웹 음성 제어 AI Agent (2025.06 ~ 2025.08)
문제정의: 사용자가 음성 명령만으로 웹을 탐색·제어할 수 있는 AI 인터페이스 구축 필요.
역할: STT(Whisper) + LLM(OpenAI) + Playwright 연동을 통한 자연어웹 제어 에이전트 구현
기술: 명령 파싱/동작 매핑, 브라우저 자동화, 모듈형 에이전트 아키텍처
주요기여: 음성 인식 결과를 LLM 프롬프트로 변환해 페이지 이동·정렬·오디오 제어 등 자동화 기능 설계.
성과/피드백: 프롬프트 설계가 제품 UX와 안정성에 직접적 영향을 미침을 검증, AI 에이전트 설계 시 사용자 경험 중심 사고의 중요성을 체득.

4) 케어키즈 – 리뷰 기반 장소 추천(NLP) (2024.08 ~ 2025.01)
문제정의: 아이를 둔 부모에게 리뷰 데이터를 기반으로 신뢰도 높은 장소 추천을 제공하고자 함.
역할: 리뷰 크롤링전처리감성분석추천까지 NLP 파이프라인 구축
기술: Selenium, BERT 파인튜닝, Word2Vec, 텍스트 정규화/불용어 처리
주요기여: ko-BERT 모델을 직접 파인튜닝하여 정확도 56% 95% 향상, 긍정 리뷰 키워드 + Word2Vec 유사도로 아이 동반 친화 장소 추천 모델 완성.
성과/피드백: 데이터 수집부터 모델링·서빙까지 전 과정을 직접 경험하며, NLP 서비스의 실무 적용성과 모델 해석력의 중요성을 배움.

[입사 지원 동기 및 포부]
저는 “AI로 더 나은 세상을 만든다”는 목표로 학부 시절 CNN·YOLO 프로젝트로 기초를 다지고, KT AIVLE School에서 NLP·시계열로 영역을 확장했습니다. 현
재는 IoT 음향 기반 누수 판별 CNN, 발전소 고소음 환경 적응화, 음성 제어 LLM 에이전트를 개발하며 데이터 수집–전처리–모델링–서비스 구현(E2E) 전 과정을
경험했습니다. 이 과정에서 현장 제약·사용자 니즈를 반영한 설계가 성능만큼이나 중요하다는 것을 깊이 이해했습니다. 제 경험과 기술을 바탕으로 AI 모델 개발·
데이터 분석·서비스화까지 유기적으로 연결하여 신뢰성과 사용성을 갖춘 제품을 만드는 데 기여하겠습니다. 나아가 축적되는 데이터를 기반으로 지속 개선이 가
능한 파이프라인을 구축해, 사용자와 사업이 동시에 체감하는 가치를 창출하는 데 적극 참여하겠습니다.
